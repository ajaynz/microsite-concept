Market Signal

Everyone is racing to build a software factory. Not buying one. Building one

A factory is infrastructure. It’s process. It’s a system that runs reliably at scale. And increasingly, autonomous coding agents that generate code, review, test, and merge pull requests around the clock.

Minions merges over 1,300 AI generated pull requests every week. Background coding agents go from a Slack message to a CI-passing PR with no human in the loop. Engineers tag the Minion bot in Slack and walk away. The agent runs in an isolated pre-warmed devbox, executes the task, and opens a PR.

Honk has shipped more than 1,500 agent-authored PRs into production. Spotify adapted its fleet management system into a platform for running a background coding agent that performs large-scale code migrations from natural language instructions.

Inspect now authors roughly 30% of Ramp’s merged pull requests. It runs inside sandboxed cloud VMs with full access to the real development environment, from Postgres and Redis to Temporal, LaunchDarkly, and Buildkite.

Minion has 84% of Uber developers using agentic tooling monthly. Uber also built an internal MCP gateway that exposes source code, Jira, Slack, and engineering documentation as a unified context surface for agents.

Agent Executes

Agent starts executing with precise context.

Does the agent have the correct context?

A good answer depends on more than file access. It depends on reaching the real implementation, understanding impact, pulling the right history, and showing its work.

Audit target Correct Context

Real implementation, blast radius, evidence, and team history aligned before reasoning begins.

01

Canonical implementation

If the logic lives in a shared service or behind an interface, the agent still needs the real source of truth.

Find the actual implementation first.

02

Blast radius

The system should trace callers, downstream consumers, and migration risk across repository boundaries.

File access alone stops being enough here.

03

Evidence trail

A trustworthy answer points back to paths, symbols, references, or history instead of feeling magical.

A reviewer should be able to inspect how it knew.

04

Precision, not volume

The agent should surface the few artifacts that explain prior intent, ownership, and the existing team pattern.

Too much context is another form of failure.
